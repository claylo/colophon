## Where things stand

Colophon is release-ready. This session addressed all findings from the 2026-03-21 Codex audit (`record/audits/2026-03-21-current-repo-review/`), added char-boundary panic guards for the Unicode casefold issue, and fixed the dominant source of phantom locations (29 → 1). The incremental curate pipeline was smoke-tested end-to-end against the real 67-file Typst corpus with actual Claude invocation — it works, costs ~$4.57, and produced a sensible conservative delta.

## Decisions made

- **Validation always reports** — `display_validation()` triggers on `unresolved > 0`, not just when suggestions exist. Unresolved locations without suggestions now print to stderr. (PR #16)
- **Path::ends_with for main_files** — replaced `contains()` substring match to prevent `auth.md` from falsely matching `old-auth.md`. (PR #16)
- **Char-boundary guards, not full Unicode rewrite** — added `is_char_boundary()` checks at `insert_str`, `find_term_offset`, and `find_term_offset_in_prose`. Terms with drifted offsets become "not found" instead of panicking. Full correctness fix (finding #3) deferred. (PR #16)
- **Auto-alias known_terms variants** — `post_process` now adds config's `known_terms` variants as aliases, matching by curated term name OR by alias. This handles both direct matches and Claude rename chains. (PR #17)
- **curate::run() takes &[KnownTerm]** — public API change to thread known_terms into post_process without exposing the full config.
- **Audit remediation ledger** — `record/audits/2026-03-21-current-repo-review/actions-taken.md` tracks all three findings as fixed/mitigated.

## Landmines

- **Existing terms files don't have the new aliases.** The auto-alias fix runs at curate time. The 67-file corpus at `~/source/claude/build` still has the old `colophon-terms.yaml` without variants. Next full or incremental curate will pick them up.
- **Incremental curate at 80% new is expensive.** First incremental run after a full curate shows ~80% "new" candidates because curate renames heavily. Cost is nearly the same as full rebuild (~$4.57 vs ~$5-8). Real savings come on subsequent runs with small corpus changes.
- **`serde_yaml` is deprecated upstream.** Functional, but `Cargo.lock` shows `serde_yaml 0.9.34+deprecated`. Not blocking but worth migrating eventually.
- **The `.justfile` is a dotfile.** Release readiness sweep missed it because it looked for `Justfile`/`justfile`. It exists as `.justfile`.
