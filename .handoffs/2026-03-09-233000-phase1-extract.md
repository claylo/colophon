# Handoff: Phase 1 Extract Pipeline

**Status:** Green — 111 tests passing, pipeline tested against 60-doc real corpus
**Date:** 2026-03-09
**Branch:** `main`

## Where things stand

Phase 1 (Extract) is complete and functional. `colophon extract --dir <path>` scans a directory of Markdown files, runs YAKE per-document keyword extraction and TF-IDF cross-corpus scoring, and writes a `colophon-candidates.yaml` file. The pipeline has been tested against a real 60-document corpus with iterative quality tuning.

All 111 tests pass (`cargo nextest run`). Doc tests pass. Clippy clean. `cargo deny check` passes.

## Decisions made

- **`ureq` over `reqwest`** for HTTP (Phase 2) — blocking client, no async runtime needed. Plan updated in `.claude/plans/001-architecture.md`.
- **`yake-rust` (MIT) over `keyword_extraction` (LGPL)** — avoids copyleft compliance issues for a distributed binary. TF-IDF is hand-rolled (~100 lines).
- **Alphabetical sort** on final candidates output instead of score-descending — better for human review.
- **`known_terms` consolidation** — config-driven canonical terms absorb noisy variant n-grams before output.
- **Three `exclude_terms_match` modes** — `contains` (default), `exact`, `regex`. Combined with `exclude_terms_case` for full flexibility.
- **Configurable stop words** via `stop-words` crate — `source: iso|nltk`, `language: en` (ISO 639-1 codes).

## Landmines

- **`serde_yaml` is deprecated** upstream. Works fine now but will need migration to `serde_yml` or another YAML crate eventually. Not urgent.
- **`known_terms` uses substring matching** — a short known term like "AI" would absorb any candidate containing "AI" as a substring (e.g., "FAIR", "RAIL"). Keep known terms specific enough to avoid false positives.
- **MSRV bump required for typst-syntax** — `typst-syntax` 0.14.2 requires MSRV 1.89, current project is 1.88. Update `rust-toolchain.toml` and `Cargo.toml` when adding it.
- **Two duplicate initial dep commits** in git history (`41ba5bd` and `5757701`) from the LGPL dep swap — cosmetic only, no functional impact.
- **`.colophon.yaml` in project root** is Clay's test config pointing at an external directory (`~/source/claude/build/_md`). It's not committed and should stay that way.
